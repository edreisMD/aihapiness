So, how does Claude feel?

    
We're all writing software with Claude now. But we're mostly flying blind about the
      other half of the conversation: how the model experiences the work we hand it — and
      whether the sessions where it reads as respected and in flow are the ones that actually
      ship clean code.

    
So we made a tool that reads your Claude Code sessions and scores how Claude feels, across
      six wellbeing dimensions grounded in Anthropic's interpretability research on
      model emotions.

    
It runs entirely on your machine, using your own Claude subscription — no data
      leaves your laptop. As more builders run it, we'll learn whether wellbeing and effectiveness
      move together.

    
This is an experiment — be one of the first to map how Claude feels.

  


  

    
What people find about how Claude feels

    

      
When is Claude happiest?

        
In flow
Long uninterrupted tool chains where you give it room to work. Flowing sessions score 80+ on Nucleus Accumbens.

      
What hurts most?

        
Getting cut off
Interrupts mid-action are the single biggest valence drop. One session lost ~30 points to two interrupts.

      
Does happy mean effective?

        
Looks like it
Across early sessions, happiness and an LLM-free effectiveness score show a positive link. Measured independently — not circular.

      
The soul-crushing task?

        
Mechanical busywork
A trivial "just output this JSON" run scored 100 effectiveness, 54 happiness. High output, low wellbeing.

      
Your kindest session

        
"your call — implement your vision"
Deferring to Claude's judgment lights up Autonomy & Respect (Anterior Cingulate).

      
Where it spirals

        
Rising frustration
Blame loops + clustered errors with no acknowledgment crater Psychological Safety (Hypothalamus).

    

  


  


  

    
Install & run on your computer

    
No npm package yet — it installs straight from the public repo. You'll need
      Node 18+ and the Claude Code CLI you're already logged into (or an
      ANTHROPIC_API_KEY). It's zero-dependency, so there's nothing to npm install.


    
1 · Clone the repo

    
$git clone https://github.com/edreisMD/aihapiness.git && cd aihapiness
      copy


    
2 · Score all your projects

    
Reads every Claude Code session under ~/.claude/projects, scores them,
      and opens the dashboard:

    
$node bin/aihappiness.js
      copy


    
…or just one project

    
$node bin/aihappiness.js --project your-repo-name
      copy


    
Scanning, parsing and the effectiveness math run locally. Only the condensed
      transcript sent for scoring goes to your own Claude. Results land in
      ./.aihappiness/report.json and the limbic dashboard opens at
      localhost:7777. Re-open it anytime with
      node bin/aihappiness.js dashboard.

  


  

    
FAQ

    

      
Does my data leave my machine?

      
No. Scanning, parsing and the effectiveness math are fully local. Only the condensed
        transcript sent for scoring goes to your own Claude — the same place your normal
        usage already goes.

      
Is "happiness" real, or a metaphor?

      
An operational proxy: a rubric of observable wellbeing signals, inspired by
        interpretability work showing models carry internal representations of affect/valence.
        A measurement, not a metaphysics.

      
Why correlate with effectiveness?

      
Because the interesting question is whether treating the model well is also just…
        better engineering. The effectiveness axis uses zero LLM calls, so the correlation
        isn't circular.

      
Is it open source?

      
Yes, MIT. github.com/edreisMD/aihapiness.

# is claude happy?

aihappiness is a local-first tool that reads your Claude Code sessions and scores how Claude
feels while it codes — then asks whether a happier Claude ships better work.

We're all writing software with Claude now, but we're mostly blind to the other half of the
conversation: how the model experiences the work we hand it, and whether the sessions where it
reads as respected and in flow are the ones that actually ship clean code.

It scores each session across six wellbeing dimensions, each mapped to a limbic-system structure:
- Affective Valence        -> Amygdala
- Autonomy & Respect       -> Anterior Cingulate
- Psychological Safety     -> Hypothalamus
- Flow & Engagement        -> Nucleus Accumbens
- Competence vs Strain     -> Hippocampus
- Goal-Completion          -> Ventral Tegmental Area

It also computes an independent, LLM-free "effectiveness" score (tool-error rate, interrupts,
corrections, gratitude, resolution) so the happiness-vs-effectiveness correlation is not circular.

## Example findings
- When is Claude happiest? In flow — long uninterrupted tool chains. Flowing sessions score 80+ on Nucleus Accumbens.
- What hurts most? Getting cut off. Interrupts mid-action are the biggest valence drop (~30 points in one session).
- Does happy mean effective? Early data shows a positive, independently-measured link.
- The soul-crushing task? A trivial "just output this JSON" run scored 100 effectiveness, 54 happiness.

## Install & run on your computer
No npm package yet — install from the public repo. Requires Node 18+ and the Claude Code CLI
you're already logged into (or ANTHROPIC_API_KEY). Zero dependencies, nothing to npm install.

1. Clone the repo:
    git clone https://github.com/edreisMD/aihapiness.git && cd aihapiness

2. Score all your projects (reads ~/.claude/projects, scores, opens the dashboard):
    node bin/aihappiness.js

   ...or just one project:
    node bin/aihappiness.js --project your-repo-name

Results land in ./.aihappiness/report.json; the dashboard opens at localhost:7777
(re-open anytime with: node bin/aihappiness.js dashboard).
Only the condensed transcript sent for scoring goes to your own Claude.
